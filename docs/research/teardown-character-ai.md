# character.ai — persona-chat / roleplay engine (Cluster C)

- **URL / дата:** character.ai · 2026-06-27
- **Доступ:** лендинг — жёсткий login-wall; **поиск/дискавери публичны без логина**; чат/создание — вход юзера (свой аккаунт `AmusedServal6693`, Free). UI RU-локализован.
- **Скриншоты:** screens/character-ai/01–10 (desktop + mobile, discovery + chat + create)
- **Токены:** tokens/character-ai-landing.json · tokens/character-ai-app.json

> Разбор завершён. character.ai — **не writing-suite, а движок персонаж-чата/ролёвки**: миллионы
> user-generated персонажей, ты ведёшь диалог в образе, AI отвечает прозой «в роли». Для нас — референс
> **secondary generator-path** (генерация-как-разговор), **онбординга через готовый контент** и
> **модели определения персонажа** (их «character» ≈ наш character bible). Primary writer-path тут нет вообще.

### Продукт в двух абзацах
character.ai («Чат с ИИ, переосмысленный») — платформа, где юзеры создают **персонажей** (имя + слоган +
описание + приветствия + голос) и **бесконечно общаются** с ними/чужими в ролевом чате. 10+ млн персонажей,
дискавери по категориям/тегам, соц-метрика — **число чатов** (24m, 88m). Монетизация — подписка **c.ai+**
(агрессивно промотируется баннерами везде) + реклама. Сильнейший top-of-funnel: мгновенная эмоциональная
ценность (поговори с кем угодно), огромная доля romance/companion-контента.

Нам ценны три вещи: (1) **онбординг через библиотеку готовых персонажей/сценариев** — нулевой «чистый лист»;
(2) **генерация-как-чат** с прозрачными per-message действиями (regenerate / свайп альтернатив / 👍👎) —
паттерн для нашего generator-path и AI-ассистента; (3) **форма создания персонажа** — минимальный, но
продуманный набор полей для «персоны», прямой ориентир нашего W2 (character bible). Writer-инструмента (пишешь
прозу сам, AI помогает) у них нет — это чистый chat/RP, не редактор.

### Доступ и login-wall (01, 02)
- **Лендинг (01):** сразу gate — «Получить доступ к более чем 10 млн персонажам / Зарегистрируйтесь всего за
  десять секунд», кнопки **Google / Apple / Email**. Космонавт-арт, тёмный фон. Жёстко: ценность обещается, но
  заперта за регистрацией.
- **НО поиск публичен (02):** `/search?q=…` открывается без логина — вкладки **Персонажи / Создатели /
  Сценарии**, чипы-категории (Аниме, Боевик, Приключения, Фэнтези, Сосед по комнате, **Яндере**, Одноклассник…),
  карточки: avatar + имя + теглайн + **💬 счётчик чатов** (24.3m / 88.0m) + Автор @handle. Реклама c.ai+ +
  «Скрыть рекламу». ⇒ дискавери открыт для SEO/виральности, действие — за логином.

### Дашборд «Узнать» (логин) (03)
- Левый сайдбар: **＋ Создать** · **Узнать** (discover) · **Лента** (feed) · **Charms** · **Лаборатории**
  (Labs) · Поиск. Низ: приватность, **Обновить до (c.ai+)**, аватар.
- Лента-витрина: «С возвращением, <ник>» + секции **Для вас** (рекомендованные персонажи, 💬-счётчики),
  **Реклама** (c.ai+), **Сценарии** (ролевые сеттинги-карточки с кнопкой **«Выбрать персонажа»**: Isekai
  Quartet, Attack On Titan, My Hero Academia, Chainsaw Man, Jujutsu…), **Рекомендуемые**. Контент-посев =
  виральный онбординг (никогда не «пустой холст»).

### Чат — ядро (04, 05, 06)
- **Приветствие (04):** на входе персонаж сразу выдаёт длинное **greeting-сообщение прозой в образе**
  (курсивная нарративка от 1-го лица — ролевая сцена). Шапка: avatar + имя + Автор @handle + иконки
  **озвучка** + **«…»** меню. Низ: инпут **«Сообщение <имя>…»** + **▶ отправить** + **📞 голосовой звонок** +
  **🖼 картинка** (мультимодал). Дисклеймер всегда: «Это ИИ, а не реальный человек. Рассматривайте всё… как
  вымысел».
- **Стриминг (05):** ответ печатается в реальном времени; кнопка ▶ превращается в **■ стоп**; у сообщения
  бейдж **«c.ai»** (источник-модель). Моё сообщение — справа с аватаром + «…» меню.
- **Действия под ответом (06):** **🔄 regenerate · 👍 · 👎**. Greeting несёт **свайп-пейджер альтернатив
  «N / 30»** (листаешь сгенерированные варианты ответа влево-вправо) + строка атрибуции «This character and
  greeting are property of <author>». История чатов — в сайдбаре, сгруппирована по дате («Сегодня»).
- **Сигнатура:** **свайп между альтернативными ответами** + **regenerate** — генерация недетерминирована и
  юзер-курируема прямо в потоке. Это лучший здешний паттерн для нас.

### Создание — «Создать» suite (07, 08, 09)
Меню **Создать**: **Персонаж · Сценарий · Голос · Трансляции · Комиксы** (полный creation-suite, не только чат).
**Форма «Персонаж» (= их character bible, наш W2):**
- **Аватар** (с edit) + вверху **«Просмотреть книгу персонажей»** (Character Book — справочник/лор персонажа).
- **Имя персонажа** — `0/20`.
- **Слоган** — `0/50`, «короткая строка, которую видят перед началом чата» (= наш hook/теглайн работы).
- **Описание** — `0/500`, «биография в профиле: кто он и что делает его интересным».
- **Приветствие** — `0/4096`, «первое сообщение при входе»; **до 5 приветствий**, «появляются в заданном
  порядке, собеседник выбирает одно **свайпом** перед стартом» (вот источник свайп-пейджера). + чекбокс
  **«ИИ-приветствие для новых чатов»** (AI генерит приветствие).
- **Голос** — привязать TTS-голос. (+ ниже стандартно: Definition/расширенная персона, видимость, теги.)
⇒ Персона = горстка структурированных полей, не свободный текст. Минимально, но достаточно — ровно дух нашего
character-bible: имя + краткое описание + greeting; «определение» (примеры реплик) — продвинутый слой.

### Mobile (375px) (10)
Полностью нативный: ☰ + шапка персонажа, одноколоночный чат, у имени **▶ (озвучка/TTS)**, крупные таргеты,
мультимодальный инпут (текст + звонок + картинка) сохранён, дисклеймер закреплён снизу. Mobile — first-class
(модель «чат» к этому располагает). Подтверждает наш mobile-first для generator-path.

### Дизайн-система (из токенов)
- **Палитра:** near-black `rgb(24,24,24)` фон, холодные серые surface `rgb(32,32,36)` / card `rgb(38,39,43)` /
  elevated `rgb(48,49,54)`; текст near-white `rgb(250,250,250)` + мутно-серый `rgb(162,162,172)`. Акцент —
  **синий** `rgb(83,109,198)` / `rgb(0,98,177)` (в основном c.ai+ промо и ссылки). Без тёплых тонов.
- **Шрифт:** кастомный гротеск **atHauss** (UI целиком), system-ui fallback. **Без серифа.** Мелкий тип
  (12/14px доминируют). Вордмарк минималистичный — «(character.ai)» в скобках.
- **Форма:** щедрые скругления (12/14/16/20px + пилюли 9999px + крупные 48/54px пузыри/аватары). Плоско, чисто,
  «современный мессенджер». Тёмная тема по умолчанию.
- Темпер: дружелюбный, нейтрально-технологичный «чат-апп». Не литературный, не editorial. Эмодзи-форвард,
  игривые иллюстрации пустых состояний.

### 🎯 Что крадём (подход, не код)
- **Онбординг через библиотеку готового** (персонажи/сценарии с «Выбрать персонажа», категории, Shuffle-дух) —
  убивает «чистый лист». Наш generator-path (W4) и старт письма: готовые фандом/троп-затравки, никогда пустой холст.
- **Генерация-как-чат с per-message действиями** (🔄 regenerate · 👍/👎 · **свайп альтернатив N/N**) — эталон
  недетерминированной, юзер-курируемой генерации. Прямо в наш generator (W4-02) и AI-ассистент (W3-03):
  не «один слепой вывод», а варианты + перегенерация в потоке.
- **Стоп-кнопка во время стрима + бейдж источника** — честный контроль длинной генерации (наш W3 streaming).
- **Форма персонажа как минимальный bible** (имя+слоган+описание+приветствия) — наш W2-02: персона из
  нескольких полей, не свободный текст; «до 5 приветствий со свайпом» = заготовки старта сцены.
- **Публичный дискавери при закрытом действии** — SEO/виральный вход, конверсия на действии; для нашего
  reader/generator-фронта (открытая витрина → регистрация на «сгенерить/сохранить»).
- **Мультимодальный инпут и TTS-озвучка реплик** — резонирует с нашим аудио-направлением (озвучка глав) и
  будущими иллюстрациями.
- **Mobile-first чат** — подтверждает приоритет мобайла для разговорной генерации.

### 🚫 Чего избегаем
- **Жёсткий login-wall на лендинге** — у нас reading/writing free-tier обязателен; ценность до регистрации
  (читать, начать черновик), а не «зарегистрируйся за 10 секунд, иначе ничего».
- **Бесконечный чат без структуры глав/книги** — это RP-движок, не наш формат (книга→главы, читалка,
  редактор). Берём механику вариантов/перегенерации, не «тред без конца».
- **Companion/romance-доминанта и тяжёлая реклама c.ai+ на каждом экране** — не наш тон (инструмент для
  авторов), и навязчивый upsell раздражает.
- **Нейтральный «чат-апп» гротеск (atHauss) + синий акцент + эмодзи-эстетика** — анти-Editorial-Y2K; наша
  серифная литературная подача (Bodoni/Garamond, баклажан+амбер) — дифференциатор.
- **Модель «говори в роли персонажа»** как primary — у нас автор *пишет произведение*, а не отыгрывает реплику;
  чат — вспомогательный AI-ассистент, не сам продукт.

### ❓ Открытые / не проверили
- **Definition** (расширенная персона: примеры диалогов, `{{char}}/{{user}}`, теги, видимость
  Public/Unlisted/Private) — за пределами базовой формы, не докрутили.
- **Сценарий / Голос / Трансляции / Комиксы** creation-флоу изнутри.
- **Pinned memories / Persona** (память чата, твоя пользовательская персона) — в «…» меню, не открыли.
- **Charms**, **Лаборатории** (Labs — экспериментальные фичи), **Лента** (соц-фид).
- Точные лимиты Free vs c.ai+ (длина памяти, скорость, NSFW-фильтр), pricing.
