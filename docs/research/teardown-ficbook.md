# Ficbook (Книга Фанфиков) — RU fanfic platform, наш прямой бенчмарк (Cluster B)

- **URL / дата:** ficbook.net · 2026-06-27
- **Доступ:** Cloudflare Turnstile блокировал авто-браузер → юзер прошла сама и прислала **26 скриншотов** (свой залогиненный аккаунт + публичные страницы-визитёра). Этично: смотрим то, что видит любой визитёр + свои экраны, учимся подходу.
- **Скриншоты:** screens/ficbook/01–26 (desktop + mobile, reader + writer + settings)
- **Токены:** tokens/ficbook-home.json (снят до Cloudflare-блока; остальные страницы — без token-grab, т.к. шли через скрины юзера)

> Разбор завершён. Ficbook — **самый важный референс Cluster B**: это ровно наша ниша (RU-фанфик,
> Фикбук-сегмент). Не AI-инструмент — это **зрелая two-sided платформа автор↔читатель** с глубочайшей
> tag-таксономией, ролевой моделью (автор/читатель/помощник) и монетизацией через промо/премиум/монеты.
> Редактор у них — голый CMS без AI. Здесь наш product-gap и наша возможность.

### Продукт в двух абзацах
Ficbook — крупнейшая русскоязычная платформа фанфиков (и ориджиналов): читатели находят/читают, авторы
публикуют главами, между ними — плотный социальный слой (отзывы, подписки, сборники, подарки, награды,
заявки). Контент-дискавери построен на **направленности** (тип отношений) + **рейтинге** + **метках**
(жанры/предупреждения/тропы/формат) с миллионными счётчиками. Тёмная тема, плотный «приложенческий» UI.

Для нас это **эталон reader-path и tag-системы** (наш W2/discovery) и **демонстрация того, чего у фикбука
нет**: AI-ассистента для письма. Их writer — простой rich-text редактор «по частям» с метадата-шапкой, ноль
интеллекта. Headcanon входит ровно сюда: «Фикбук, но с AI-соавтором + современный editorial-UI». Берём их
проверенную discovery/social-механику, отвергаем их голый редактор и плотную утилитарную эстетику.

### Дискавери: направленность + рейтинг + метки (ядро платформы)
Три ортогональных измерения, по которым фильтруется/маркируется ВСЁ:

**1. Направленность** (тип отношений, у каждой своя иконка-глиф):
Джен · Гет · Слэш · Фемслэш · Другие виды отношений · Смешанная · Статья. — first-class фильтр, не «метка».

**2. Рейтинг:** G · PG-13 · R · NC-17 · NC-21 (color-coded бейдж на каждой карточке).

### Карточка работы (плотнейшая метадата — 03/05/07/11)
Заголовок → бейдж-ряд: **направленность** (цветной) + **рейтинг** + **статус** (В процессе/Завершён) +
**👍лайки** + **🏆награды** + **«Горячая работа»**. Затем:
- **Автор** (avatar + ник, 👑 если топ/премиум)
- **Фэндом** (напр. Jujutsu Kaisen)
- **Пэйринг и персонажи** — с нотацией ролей: `альфа!Сатору Годжо/омега!Сугуру Гето`, `Топ!…`, `Боттом!…`
- **Размер:** `планируется Макси, написана 91 страница, 31 849 слов, 6 частей` — план + факт в страницах/словах/частях
- **Метки:** облако (с 18+ маркерами), переполнение → «Спойлеры …»
- **Описание** (синопсис) + обложка-превью (2:3, атмосферный AI-арт)

→ Богатая «шапка» — отдельный слой ПЕРЕД чтением. Читатель решает по метаданным.

### Reader-path (12–14, 16)
- **Содержание (12):** Промо-карусель чужих работ сверху + список глав. Каждая глава:
  заголовок + дата + **«Отзывы: N» на каждую главу** (engagement-гранулярность по главам) + chevron.
  Главы названы авторски («Предисловие. Вход в подвал», «Глава 3 — Синдром отмены»).
- **Reader-view (13):** тёмный фон, комфортная колонка, текст по ширине (justify-наклон). Минимальный хром.
  Низ-навигация **Содержание / Вперёд ›**. Авторские эпиграфы/примечания внутри текста.
- **Action-бар работы (13):** 👍 Нравится · ⭐ Подписаться · 💬 Отзывы · 📖 В сборник · ☑ Прочитано · 🚩 ·
  «N сборников». — лайк ≠ подписка ≠ закладка-в-сборник ≠ «прочитано»: четыре разных сигнала.
- **Отзывы (14):** «Зарегистрируйтесь или войдите чтобы написать» + соц-логин **VK / Yandex / Apple**.
  Комменты: avatar + ник + дата + текст (эмодзи) + **🏆 Наградить** (выдать награду комменту!) + 👍.
  Threaded: автор отвечает цитатой; у автора 👑 + бейдж «автор». Сортировка «Сначала старые».
- **Mobile (15, 16):** полностью адаптивно. Шапка работы стекается в колонку, облако меток переносится,
  reader-view на всю ширину — комфортно. Mobile — гражданин первого класса.

### Заявки — demand-side marketplace (10)
«**Заявка — это оформленная читателем идея для фанфика, по которой любой автор может написать работу.**»
Вкладки Всё / Горячие 2127 / Популярные. Карточка заявки: заголовок + «Горячая заявка» + **👍104 · ⭐57 ·
📄0** (лайки / в избранном / откликов-работ). Поля: **Фэндомы · Направленности · Рейтинг · Метки · описание.**
Фильтр-сайдбар: Тип (Ориджиналы/Фанфики) · Только с написанными работами · Фэндомы · Метки · Рейтинг ·
Направленность · Популярность заявки · Дата · Ключевые слова + «Добавить заявку». — **читатели формируют
спрос, авторы его удовлетворяют**; та же метадата-схема, что у работ. Прямая two-sided-механика.

### Социальный слой и геймификация (08, 09, 14)
- **Авторы-лидерборд (08):** «Популярные авторы» №1…N, avatar + ник + 👑, «+208 подписчиков», вкладки
  **За 7 дней / За всё время**. Рост подписчиков = метрика ранжирования.
- **Профиль автора (09):** табы Информация / Блог / Работы (N) / **Подарки** / **Награды** / Отзывы /
  **Достижения**. Кнопки **🎁 Подарок** + ✉. «О себе» + Telegram-линк.
  - **Подарки:** «25 подарков» + «Сделать подарок» — каталог виртуальных AI-арт стикеров (перо, золотая
    книга, котики, единорог, цветы…). Дарят друг другу.
  - **Награды:** «51 награда работы» 🏆.
- **Награждение контента:** награду можно повесить и на работу, и на **отдельный комментарий** (14).
- **Монеты (22):** «Купить монеты (0 монет)» — виртуальная валюта (под подарки/промо).

### Ролевая модель — три кабинета в одном аккаунте (21, 22)
Дропдаун аватара чётко разделяет роли одного юзера:
- **Кабинет автора:** Добавить фанфик · Мои фанфики · Мой блог · Отзывы · История изменений ·
  Сообщения об ошибках · Персональный баннер.
- **Кабинет читателя:** Сборники · Закладки · Обсуждения · Подписки на авторов · Понравившиеся работы ·
  Прочитанные работы · История подписок · Просмотренные работы · Мои ленты.
- **Кабинет помощника:** Анкета помощника · Фанфики помощника. — **бета-ридер/соавтор как отдельная роль**
  с анкетой-профилем.
+ Личные сообщения, Заявки, Связь, Настройки, Купить монеты. → один человек = автор + читатель + помощник,
переключается ролями. Прямо релевантно нашей writer+reader-двусторонности.

### Writer-path — голый CMS-редактор (17, 18). ⚠️ Наш главный gap
**Метадата-first.** Сначала «**Заполнить шапку работы**» (17): Название · **Обложка** (модерация: запрещены
копирайт/сексуальные/шокирующие/алкоголь-табак-наркотики; ≤5мб, ≥600×600, jpg/png/webp; **загрузка обложки
только для Улучшенного Аккаунта или работ в Промо** → upsell) · Конкурс · **Тип** (Ориджинал / Фанфик по
фэндому) · **Рейтинг** (G…). Левый сайдбар работы: ФАНФИК (Шапка · Содержание · Отзывы · В сборниках ·
Статистика · Страница работы · Мои фанфики) + ПРОДВИЖЕНИЕ (Добавить в «Промо» · в «Горячее»).

**Текст-редактор (18)** — глава = «Часть N». Toolbar: *I* / **B** / ~~S~~ · align L/C/R · undo/redo · **Tab** ·
**[?]** (примечание/спойлер) · **\*\*\*** (разделитель сцен) · ❄ · **.docx** импорт · fullscreen. Плейсхолдер
«Начните писать». Низ: **Предпросмотр · Шапка · «Изменения будут сохранены»** (автосейв) · **страниц: 1, слов: 0**.
→ **Никакого AI. Нигде.** Чистый rich-text + автосейв + docx-импорт + счётчик. Вот сюда мы и заходим.

### Аналитика работы (19, 20) — местами очень умная
- **Сводка (19):** 👁 Просмотры · 💬 Отзывы · 📖 Сборники · 👍 Нравится · 🏆 Награды.
- **Просмотры:** график с диапазоном дат, фильтром «Промо», 7 дней / Месяц.
- **Активность (20):** оценки/отзывы/скачивания по дням — **premium-gated** («доступна в улучшенном аккаунте»).
- **Вовлечённость читателей (20):** **donut read-depth funnel** — Только шапка / Начало текста / 20% / 40% /
  60% / 80% / Весь текст. «Сколько дочитывают до конца, а сколько уходят после первых строк.» Premium.
  → Сильнейший паттерн: **воронка глубины дочитывания**. Прямо ложится на нашу reader-аналитику.

### Настройки и контент-контроль (23, 24)
- **Отношение к критике (23):** Не указано / приветствует всю / только в мягкой форме / не приветствует —
  **показывается у поля отзыва в каждой работе**. Соц-норма как настройка автора (здоровье комьюнити). Умно.
- Личные сообщения (вкл/выкл) · Режим невидимки · **Рейтинги: opt-in** в «Популярные фанфики» / «Популярные

- **Скрытие фанфиков (24, premium):** полностью скрыть из выдачи работы/авторов/**пейринги/направленности/
  рейтинги/метки**, которые не интересны. Tag/pairing/author **blocklist** — premium-фича.
- Безопасность: соц-логины как способы авторизации, смена ника/почты/пароля.

### Дизайн-система (наблюдения + tokens/ficbook-home.json)

- **18+ маркеры / цветные рейтинг-бейджи / иконки направленности** — продуманная визуальная система
  предупреждений (берём принцип, не вид).
- Темпер: «огромный функциональный портал для своих». Мощно, но визуально тяжело и старомодно.

### 🎯 Что крадём (подход, не код/ассеты)
- **Три измерения дискавери: направленность + рейтинг + метки** — проверенная схема нашей ниши. Метки с
  **категориями** (Формат/Жанры/Предупреждения/Отношения) + определениями + счётчиками. Наш discovery/tag-слой
  (W2 + читалка): берём таксономию и дефиниции как готовый словарь (см. локализацию тропов).
- **Богатая «шапка работы»** (направленность+рейтинг+статус+размер в словах/частях+пейринг+метки+синопсис) —
  наша карточка/страница работы должна нести столько же сигнала перед чтением.
- **Персонализация меток** (нравятся→зелёным / не нравятся→исключить) — дешёвый, любимый комьюнити discovery-буст.
- **Read-depth funnel** (Только шапка→Весь текст) — лучшая идея их аналитики; кладём в writer-дашборд.
- **Action-бар читателя из 4 раздельных сигналов** (лайк / подписка / в-сборник / прочитано) — чёткая
  семантика вовлечения; у нас не смешивать «лайк» и «закладку».
- **«Отношение к критике» как настройка автора у поля отзыва** — здоровье комьюнити дёшево, на уровне UI.
- **Ролевая модель автор/читатель в одном аккаунте** — валидирует нашу writer+reader-двусторонность
  (у нас без отдельного «помощника» на старте).
- **Заявки (demand-marketplace)** — мощный двусторонний движок; кандидат в後-MVP виральную механику
  (читатель заказывает → автор/генератор отвечает). Ложится и на generator-path.
- **Mobile-first reader** — подтверждает наш приоритет (их мобайл отличный).
- **Соц-логины VK / Yandex / Apple** — обязательны для RU-аудитории (не только Google).
- **Per-chapter отзывы + авторские названия глав** — гранулярный фидбэк, ложится на нашу структуру книга→главы.

### 🚫 Чего избегаем
- **Голый редактор без AI** — это ровно наш product-gap, а не образец. Наш primary = AI-соавтор в потоке письма.
- **Плотный утилитарный CMS-UI + системный гротеск** — анти-Editorial-Y2K. Наша эстетика = дифференциатор.
- **Метадата-first онбординг писателя** (сначала заполни шапку из 8 полей) — фрикшн. У нас: пиши сразу,
  метадату достраиваем потом/с AI-помощью.
- **Загрузка обложки только за премиум** — мелочно; у нас обложки/иллюстрации — часть ценности (в т.ч. AI).
- **Чрезмерная геймификация** (подарки-стикеры, монеты, награды на комменты) — для нас это пост-MVP и в
  меру; не строим экономику стикеров с первого дня.
- **Premium-гейтинг базовой аналитики** (даже read-depth за деньги) — базовую вовлечённость даём бесплатно;
  платим за AI, не за свой же дашборд.
- **Лидерборды-по-умолчанию** — рейтинги авторов могут токсично давить; у нас opt-in или мягче.
