## 2. Источник правды по дизайну

`docs/design_handoff_writer_v2/` — канон для всех writer-поверхностей:
- `DESIGN.md` — визуальная ДНК, токены, запреты;
- `DESIGN-writer.md` — принцип «тихий центр, выразительные края», тон AI, запреты v2;
- `README.md` — маппинг экранов на `prototypes/screens-pivot*.jsx`;
- прототипы — референс поведения, **не** код для копирования (переписываем на Next.js + Tailwind + компоненты кодбазы).

Секции канвы 01–04 (Cozy/Y2K/Manga/Mix) — архив, не реализуем. Актуальны 00, 05–09.

## 3. Этапы

### Этап A — writer-core (критический путь)

**A0 — Дизайн-фундамент v2.** Обновить `handoff/tokens.css` + preset под палитру v2 (bg `#160B22`, solid `surface #1F1230`, `panel #1A0E29`, ink-based borders), добавить светлую тему редактора «бумага» (scoped CSS-переменные, `data-theme="paper"`), ревизию кнопочного языка (пилюли weight 500, узкие паддинги, без glow-теней, Bodoni-курсив в лейблах), mono-ярлык как атом.

**A1 — Мой стол** (`screens-pivot6.jsx`: `PvDeskDesktop/Mobile/EmptyMobile`). Заменяет `app/write`:
- полка обложек 2:3 — реюз `StoryPoster`; **типографская обложка** для черновиков без арта (Bodoni-титул + mono-статус `ЧЕРНОВИК · ГЛ. 7 · 12 480 СЛ`); пустой черновик — пунктирная рамка;
- momentum-панель (боль №1 аудитории — прокрастинация): стрик «N ночей подряд», спарклайн слов за 14 дней, «последнее редактирование», литературная подводка (шаблонная, без LLM — «Три ночи назад ты оставила …, гл. N ждёт») + CTA продолжить;
- пустой стол = guided start (фандом-чипы + один CTA), не blank canvas.

Данные: новая модель `WritingStat` (`userId`, `date`, `wordsAdded`, `@@unique([userId, date])`); autosave-роут при сохранении главы пишет положительную дельту слов. Стрик и спарклайн считаются кодом из последних 14–30 строк.

**A2 — Тихий редактор** (`screens-pivot5.jsx`: `PvQuietEditor*`). Рестайл существующего TipTap:
- колонка Garamond max-620px, 17–18px/1.62, ragged-right при письме;
- 3 режима: **письмо / фокус** (неактивные абзацы opacity .3 + радиальное затемнение) / **печатная машинка** (активный абзац по центру вертикали); переключатель внизу; chrome тает до opacity .16;
- светлая тема «бумага» (переключатель);
- автосейв-индикатор — mono-ярлык «сохранено» с амбер-точкой, никаких тостов;
- запреты: никакого зерна/стикеров/glow поверх текста; свет свечи — только тонкая inset-виньетка.

**A3 — Библия персонажей** (`screens-pivot2.jsx`: `PvBibleDesktop`):
- CRUD UI поверх существующей модели `Character`: карточка = маленькая обложка-градиент + Bodoni-имя + mono-теги роли; несколько структурных полей (имя, роль, затравка, описание) — не лорбук со свободным текстом;
- колонка «✦ найдено в тексте»: AI-извлечённые факты (реюз/расширение `bible_extract`) с accept/reject «в библию ✓ / мимо»;
- API: `api/write/story/[id]/characters` CRUD (auth + ownership как в существующих write-роутах).

**A4 — Граф связей** (сигнатурная фича; `screens-pivot7.jsx`: `PvGraphCore`):
- новая модель `CharacterRelation`: `storyId`, `fromId`, `toId`, `kind` (enum: ROMANCE / ENMITY / FAMILY / FRIENDSHIP / MENTORSHIP / OTHER), `label` (подпись Bodoni-курсивом), `isPast` (пунктир), `source` (MANUAL / AI), `status` (SUGGESTED / ACCEPTED / REJECTED);
- AI предлагает связи из текста глав (structured output, новый callType `relation_suggest`, через pg-boss job по образцу `bible_extract`) с accept/reject; ручное добавление тоже есть — «граф сам собирается, пока пишешь»;
- рендер: **d3-force + SVG** (не react-force-graph/sigma — ≤30 узлов, нужен полный контроль ДНК: свечной glow-«дыхание», rose = чувство / amber = сюжет, выделение по клику, `prefers-reduced-motion`);
- desktop: полотно + легенда + переключатель «граф / карточки»; mobile: упрощённая раскладка.

**A5 — Прод-инфра и деплой.** RU-VPS (Timeweb Cloud / Selectel) + Coolify + self-hosted Supabase + домен/SSL + RU LLM-агрегатор (AITunnel primary). Без прода фокус-группа невозможна. Dev остаётся на Supabase Cloud.

**A6 — Полировка.** First-run guided start (упрощённый: сразу к пустому столу A1, без развилки 3 путей — фокус-группа только авторы), empty/loading/error в литературном голосе, mobile-проход 320–1280, e2e smoke. **Запуск фокус-группы — не здесь, а после этапа B** (решение 2026-07-15: тест с первого дня включает AI-соавтора).

### Этап B — AI-соавтор (до запуска фокус-группы)

`screens-pivot8.jsx`: `PvAssistDesktop/Mobile`. По гибридной модели (Decision log 2026-06-27) и тону из `DESIGN-writer.md`:

- **B1 — чат-панель** 340px (mobile: bottom-sheet), персист в существующие `AiThread`/`AiMessage`; ожидание — «перечитываю твою сцену…» с дышащей точкой, не спиннер;
- **B2 — сборка контекста кодом**: premise + текущая глава + summary прошлых (`ChapterSummary` есть) + библия + связи графа + world state. Структурно, без парсинга свободного текста;
- **B3 — способности** (≤4 действия, человеческим языком): проверка консистентности по главам (чипы «да, исправь / так задумано»), свободные вопросы о рукописи, «что дальше» — 3 варианта хода (карточки с ♡ «оставить» / «→ в текст»);
- **B4 — inline slash-развёртка** (scene beat): затравка → прогресс-повествование → amber-pending вставка (стриминг по символам) → «в текст ✓ / иначе ↻ / мимо» + подпись «человек · AI-ассистировано»;
- **B5 — квота**: счётчик «N / M разворотов сегодня» — единственная прозрачность модели; никаких температур/выбора модели в UI. Cost-tracking через `LlmCallLog`, structured output, версии промптов — как во всей AI-инфре;
- **B6 — заметки** (глобальный inbox + per-story) — см. TASKS.md;
- **B7 — запуск фокус-группы ≤15**: инвайт-поток (модель `Invite` есть), бонусы участникам (топ при массовом запуске, 2 месяца подписки), канал сбора фидбека.

### Этап C — публичная оболочка (после валидации фокус-группой)

`screens-pivot9.jsx`, `screens-pivot2-4.jsx`: лендинг (hero «Полночное чтиво, которое пишешь ты»), первый вход с развилкой 3 путей, главная залогинена (writer-блок primary + generator secondary + лента «этой ночью читают»), кабинеты читатель/автор, бейдж «человек · AI-ассистировано» в читалке/ленте, рестайл ленты и читалки под v2-токены.

### Этап D — backlog (без изменений порядка)

Аудио с **кешем TTS по главам** (заложить поле при первом же TTS-дизайне), рублёвые тиры (ориентир PRO 199–299 ₽, STUDIO 490–690 ₽ — от якорей ниши) + YooKassa, «Заявки» (demand-marketplace), топы, видео для оригинальных миров (заморожено).

## 4. Продуктовые инварианты цикла (из ресёрча и решений)

1. **AI не пишет за автора.** Полной генерации главы в primary-пути нет; максимум — развёртка затравки с явным accept. Голос ассистента: «спрашивает, напоминает, вычитывает».
2. **Прозрачность — норма, не платная фича.** Бейдж «человек · AI-ассистировано»; «скрытие AI-generated» больше не монетизируется (решение 2026-07-03) — монетизируем качество (обложки, аудио, лимиты).
3. **Momentum — первая ценность продукта** (боль №1 = прокрастинация), не «улучшение текста».
4. **Тихий центр, выразительные края** — проверочный вопрос для каждого экрана: автор здесь пишет → тихо; выбирает/организует/любуется → можно громко.
5. **Все новые API-роуты**: auth + ownership (образец — существующие write-роуты), zod-валидация, structured output + cost-tracking + идемпотентность для AI.
6. **18+ гейтинг и tag-gating** — уже в коде, не регрессировать.

## 5. Вне скоупа этого цикла

Платежи, аудио, видео, топы, «Заявки», полноценный онбординг-развилка (упрощён до guided start в A6), рестайл generator-мастера, миграция dev-БД с Supabase Cloud.

## 6. Порядок исполнения и планы

Каждый этап получает свой implementation-план в `docs/superpowers/plans/` перед началом кода:

1. `2026-07-15-v2-a0-a1-foundation-desk.md` — **написан**, исполнять в следующей сессии.
2. A2 (редактор) — план пишется после приёмки A1.
3. A3+A4 (библия + граф) — один план, они делят Character-контекст.
4. A5+A6 (инфра + запуск) — чек-лист, TDD неприменим к инфре.
5. B1–B7 — план пишется сразу после приёмки A4 (AI-соавтор входит в MVP, запуск фокус-группы — финал этапа B).
